# Prompt Use Cases

## Debugging
- **Bug Identification**
  - **Goal**: Detect errors in code and explain why they occur  
  - **Input**: Code snippet with unexpected behavior or runtime error  
  - **Output**: Explanation of bug + corrected version  

- **Runtime Error Analysis**
  - **Goal**: Understand and fix crashes or exceptions  
  - **Input**: Stack trace or failing code  
  - **Output**: Root cause analysis + fix  

- **Logic Error Detection**
  - **Goal**: Identify incorrect program logic  
  - **Input**: Code with incorrect output but no crash  
  - **Output**: Explanation + corrected logic  

---

## Code Quality
- **Refactoring**
  - **Goal**: Improve readability and structure without changing behavior  
  - **Input**: Working but messy code  
  - **Output**: Cleaner, optimized version  

- **Code Simplification**
  - **Goal**: Reduce complexity and improve maintainability  
  - **Input**: Long or repetitive code  
  - **Output**: Shorter equivalent implementation  

- **Best Practices Enforcement**
  - **Goal**: Align code with language conventions  
  - **Input**: Code snippet  
  - **Output**: Improved version following standards  

---

## Testing
- **Test Case Generation**
  - **Goal**: Create meaningful test cases for functions  
  - **Input**: Function signature or code  
  - **Output**: Edge cases + normal cases  

- **Assertion Writing**
  - **Goal**: Validate correctness of functions  
  - **Input**: Function + expected behavior  
  - **Output**: Unit tests or assertions  

- **Edge Case Discovery**
  - **Goal**: Identify inputs that may break code  
  - **Input**: Algorithm or function  
  - **Output**: List of boundary and corner cases  

---

## Documentation
- **Code Explanation**
  - **Goal**: Explain what a function or module does  
  - **Input**: Source code  
  - **Output**: Plain-language explanation  

- **Docstring Generation**
  - **Goal**: Add proper documentation to functions  
  - **Input**: Function code  
  - **Output**: Structured docstring  

- **Technical Summarization**
  - **Goal**: Summarize code behavior for reports  
  - **Input**: Multiple functions or modules  
  - **Output**: Concise technical summary
