# User Stories

### User Story 1

As a **passenger**, I want to search for available rides near my location so that I can find affordable transportation quickly.

**Acceptance Criteria:**

* User enters pickup and destination locations.
* System displays available rides within a reasonable distance.
* Search results include driver details, price, and estimated arrival time.

**Priority:** MVP

---

### User Story 2

As a **passenger**, I want to book a ride so that I can secure transportation to my destination.

**Acceptance Criteria:**

* User can select an available ride.
* Booking confirmation is displayed immediately.
* Driver receives the booking request.

**Priority:** MVP

---

### User Story 3

As a **driver**, I want to create a ride listing so that passengers can book available seats in my vehicle.

**Acceptance Criteria:**

* Driver can enter departure location, destination, date, and available seats.
* Ride is visible to eligible passengers after publishing.
* Driver can edit or cancel the ride before departure.

**Priority:** MVP

---

### User Story 4

As a **driver**, I want to manage my ride requests so that I can accept or decline passengers.

**Acceptance Criteria:**

* Driver receives notifications for new booking requests.
* Driver can accept or reject each request.
* Passenger is notified of the driver's decision.

**Priority:** High

---

### User Story 5

As a **user**, I want to register and log in securely so that my account and personal information are protected.

**Acceptance Criteria:**

* Users can create an account using valid information.
* Registered users can log in successfully.
* Invalid login attempts display an appropriate error message.

**Priority:** MVP

---

### User Story 6

As a **passenger**, I want to pay for my ride securely so that I can complete my booking without using cash.

**Acceptance Criteria:**

* User can choose a supported payment method.
* Payment is processed securely.
* A payment confirmation is displayed after a successful transaction.

**Priority:** High

---

### User Story 7

As a **passenger**, I want to view the driver's live location so that I know when they will arrive.

**Acceptance Criteria:**

* Driver's location is updated during the trip.
* Estimated arrival time is displayed.
* Map refreshes automatically during the ride.

**Priority:** Medium

---

### User Story 8

As a **driver**, I want to view my trip history so that I can keep track of completed rides and earnings.

**Acceptance Criteria:**

* Completed trips are listed chronologically.
* Each trip includes date, destination, and earnings.
* Trip history can be accessed from the driver's dashboard.

**Priority:** Medium

---

### User Story 9

As a **user**, I want to rate and review other users after a trip so that I can help build a trustworthy community.

**Acceptance Criteria:**

* Users can submit a rating after a completed trip.
* Users can leave an optional written review.
* Average ratings are displayed on user profiles.

**Priority:** High

---

### User Story 10

As an **administrator**, I want to manage user accounts and reported issues so that the platform remains safe and reliable.

**Acceptance Criteria:**

* Administrator can view user reports.
* Administrator can suspend or restore user accounts.
* All administrative actions are recorded in an audit log.

**Priority:** Low
